# 엑셀 업로드 가이드 초안 (운영판)

## 공통 규칙

- 파일 형식: `.xlsx`만 허용
- 파일 크기: 기본 10MB 이하 (`APP_UPLOAD_MAX_FILE_SIZE_BYTES`)
- 최대 데이터 행수: 기본 5,000행 (`APP_UPLOAD_MAX_ROWS`)
- 첫 행은 헤더, 템플릿 컬럼 순서 유지
- 업로드 전 샘플 10건으로 사전 검증 권장

업로드 결과 상태:
- `SUCCESS`: 전체 성공
- `PARTIAL`: 일부 성공 + 일부 실패
- `FAILED`: 전체 실패

## 1. 부서 업로드

템플릿 컬럼:
1. 부서코드 (필수)
2. 부서명 (필수)
3. 상위부서코드 (선택)

검증 규칙:
- 부서코드 형식: 영문 대문자/숫자/`_`, 2~20자
- 같은 파일 내 중복 부서코드 불가
- 상위부서코드는 기존 부서코드여야 함

대표 실패 사례:
- 필수값 누락
- 코드 형식 오류
- 존재하지 않는 상위부서 참조

## 2. 직원 업로드

템플릿 컬럼:
1. 사원번호
2. 이름
3. 부서코드
4. 직위
5. 직책
6. 이메일
7. 로그인ID
8. 상태(ACTIVE/INACTIVE/LEAVE)

검증 규칙:
- 부서코드는 사전 등록된 부서여야 함
- 로그인ID 형식: 4~50자(영문/숫자/`._-`)
- 로그인ID/사원번호 중복 불가

대표 실패 사례:
- 존재하지 않는 부서코드
- 로그인ID 중복
- 허용되지 않은 상태값

## 3. 평가 문항 업로드

템플릿 컬럼:
1. 카테고리
2. 문항내용
3. 문항유형(`SCALE`/`DESCRIPTIVE`)
4. 최대점수 (`SCALE` 필수)
5. 정렬순서

검증 규칙:
- 문항내용 필수
- `SCALE`은 최대점수 필수
- 숫자 필드는 숫자만 허용

대표 실패 사례:
- 문항유형 오입력
- 최대점수 누락/형식 오류
- 정렬순서 숫자형식 오류

## 4. 실패 처리 표준 절차

1. 업로드 직후 화면 메시지 확인
2. `/admin/uploads/history`에서 실패 상세 확인
3. 원본 파일 수정
4. 재업로드
5. 반복 실패 시 요청 ID와 함께 운영팀 전달

## 5. 운영 팁

- 대량 파일은 분할 업로드 권장 (예: 2,000행 단위)
- 기관별 템플릿 버전을 관리해 컬럼 변경 혼선을 방지
- 신규 운영 시작 시 부서 업로드 후 직원 업로드 순서 고정
