# 운영 데이터 규모 가정 리스크 점검

## 가정

- 기관 수: 50
- 기관당 직원 수: 2,000
- 기관당 연간 평가 배정: 100,000+
- 업로드 이력 누적: 수십만 건

## 리스크 및 대응

1. 대량 조회로 인한 화면 지연
- 리스크: 업로드 이력/목록 페이지 전체 조회 시 응답 지연
- 대응: 업로드 이력 화면을 최근 N건(기본 1000)으로 제한
- 잔여 과제: 직원/세션 목록 서버 사이드 페이징 도입

2. 업로드 처리 시간 증가
- 리스크: 대용량 xlsx 업로드 시 처리 지연/메모리 사용 증가
- 대응: 파일 크기/형식 사전 검증, 실패 이력 보존
- 잔여 과제: 비동기 업로드 큐/배치 저장 최적화

3. 감사 추적 누락
- 리스크: 운영 이슈 발생 시 행위 추적 불가
- 대응: 핵심 관리자/인증/제출 이벤트 감사로그 추가
- 잔여 과제: 감사로그 조회 UI, 장기 보관 정책

4. 운영 환경 오염
- 리스크: 로컬 시드 데이터가 운영에 반영될 위험
- 대응: Flyway 경로를 `common`/`local` 분리, prod는 `common`만 사용

5. 보안 헤더/세션 설정 미흡
- 리스크: 클릭재킹/세션 탈취/보안 통제 부족
- 대응: CSP, frame-ancestors, session fixation 보호, cookie 옵션 강화

6. 배포 상태 점검 부족
- 리스크: 배포 후 장애 감지 지연
- 대응: Actuator health/info 노출, 배포 체크리스트 문서화
