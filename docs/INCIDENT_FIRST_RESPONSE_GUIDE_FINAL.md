# Incident First-Response Guide

기준일: 2026-03-11

## 1. 0~10분: 최초 인지 및 격리

1. 장애 등급 분류
- `SEV-1`: 로그인 불가/전면 장애/데이터 격리 위반 의심
- `SEV-2`: 일부 핵심 기능 장애(업로드/제출 등)
- `SEV-3`: 부분 기능 오류

2. 즉시 수행
- 영향 범위 확인(전체/기관 단위/사용자 일부)
- 최근 배포 여부 확인
- 신규 배포/작업 일시 중지

## 2. 10~30분: 1차 기술 확인

- [ ] `/actuator/health` 확인
- [ ] 앱 로그에서 requestId/예외 확인
- [ ] 감사로그에서 actor/org/action/outcome 확인
- [ ] DB 연결 상태 및 오류 확인
- [ ] 최근 환경변수/설정 변경 확인

## 3. 30~60분: 조치 결정

결정 트리:
1. 전면 영향 + 원인 불명 + 최근 배포 직후
- 롤백 우선 검토

2. 데이터 무결성/기관 격리 위협
- 쓰기 트래픽 차단 후 원인 분석

3. 국소 기능 장애
- 임시 우회 공지 + 핫픽스/설정 조정

## 5. 자주 발생 이슈 1차 대응

1. 로그인 실패 급증
- 계정 잠금/비밀번호 정책/세션 설정 점검

2. 업로드 실패 급증
- 템플릿 변경 여부, 파일 크기/행수 초과 여부 점검

3. 평가 제출 실패
- 세션 상태(IN_PROGRESS) 및 재제출 정책 확인

4. 403 증가
- 권한 변경/기관 ID 변조 시도 패턴 확인

## 6. 종료 조건

- 사용자 영향 해소
- 모니터링 정상화
- RCA/재발방지 항목 등록
