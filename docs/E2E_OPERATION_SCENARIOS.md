# 역할별 E2E 업무 시나리오 점검

## 시나리오 1. 최고 관리자 온보딩

목표: 신규 기관을 운영 가능한 상태로 만든다.

1. `ROLE_SUPER_ADMIN` 로그인
2. 기관 생성 (코드/이름/상태)
3. 기관 상세 진입 후 기관 관리자 계정 생성
4. 생성된 기관 관리자로 로그인 검증
5. 기관 상태를 `INACTIVE`로 전환 후 정책 확인
6. 감사로그 조회

검증 포인트:
- [ ] 기관 생성/관리자 생성 후 즉시 업무 수행 가능
- [ ] 기관 비활성화 정책이 의도대로 적용
- [ ] 감사로그 이벤트: `ORG_CREATE`, `ORG_ADMIN_CREATE`, `ORG_STATUS_UPDATE`

실패 시 확인:
- 중복 기관코드, 중복 로그인ID, 권한 누락 여부

## 시나리오 2. 기관 관리자 데이터 준비

목표: 평가 실행 가능한 마스터 데이터를 구축한다.

1. `ROLE_ORG_ADMIN` 로그인
2. 부서 엑셀 업로드
3. 직원 엑셀 업로드
4. 업로드 이력에서 성공/실패 상세 확인
5. 실패 행 수정 후 재업로드

검증 포인트:
- [ ] 업로드 실패에도 이력 레코드가 저장됨
- [ ] 오류 메시지와 실패 상세가 운영자가 재작업 가능한 수준
- [ ] 타 기관 데이터와 혼합되지 않음

실패 시 확인:
- 파일형식/행수/필수값/중복키/존재하지 않는 참조코드

## 시나리오 3. 기관 관리자 평가 운영

목표: 평가 세션을 생성하고 진행/종료한다.

1. 템플릿 생성 및 문항 구성(수기 또는 엑셀 업로드)
2. 세션 생성
3. 관계 자동생성
4. 예외 케이스 수동 보정(추가/삭제)
5. 세션 시작
6. 진행률 및 미제출자 모니터링
7. 세션 종료

검증 포인트:
- [ ] 시작 전/후 수정 권한 정책이 지켜짐
- [ ] 관계 생성 수량/중복 방지가 정상 동작
- [ ] 진행 현황이 실제 제출 결과와 일치
- [ ] 감사로그 이벤트가 누락 없이 기록됨

## 시나리오 4. 일반 사용자 평가 수행

목표: 실제 사용자 제출 흐름을 검증한다.

1. `ROLE_USER` 로그인
2. 본인 배정 목록 확인
3. 평가 폼 진입 후 임시저장
4. 재접속 후 저장값 복원
5. 최종 제출
6. 완료 화면 확인

검증 포인트:
- [ ] 타인 assignment 접근 차단
- [ ] 점수/서술형 유효성 위반 시 명확한 오류
- [ ] 재제출 금지 정책 위반 시 차단
- [ ] 감사로그 이벤트: `EVAL_RESPONSE_SAVE_DRAFT`, `EVAL_RESPONSE_SUBMIT`

## 시나리오 5. 기관 간 격리 공격 시뮬레이션

목표: URL 직접 접근/파라미터 변조로 데이터가 유출되지 않음을 확인한다.

1. 기관 A 관리자로 기관 B 자원 ID URL 접근
2. 세션/직원/관계/업로드/감사로그 화면 각각 시도
3. 일반 사용자로 타인의 assignment ID 접근 시도

검증 포인트:
- [ ] 403 또는 접근 차단 메시지 반환
- [ ] 데이터 본문이 노출되지 않음
- [ ] 차단 시도도 로그/감사 추적 가능

## 시나리오 6. 운영 장애 대응

목표: 장애 발생 시 운영자가 원인 추적 가능한지 확인한다.

1. 의도적으로 잘못된 업로드 파일 전송
2. 오류 화면에서 요청 ID 확인
3. 앱 로그/감사로그/DB 이력에서 동일 요청 추적

검증 포인트:
- [ ] 요청 ID 기준으로 end-to-end 추적 가능
- [ ] 재현/복구 절차(수정 후 재업로드)가 문서화되어 있음
