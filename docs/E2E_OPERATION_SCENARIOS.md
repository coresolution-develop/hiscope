# 역할별 E2E 업무 시나리오

## 1) 최고 관리자 시나리오

1. `super` 계정 로그인
2. 기관 생성
3. 기관 상세에서 기관 관리자 계정 생성
4. 기관 상태를 ACTIVE/INACTIVE 변경
5. 감사로그 확인: `ORG_CREATE`, `ORG_ADMIN_CREATE`, `ORG_STATUS_UPDATE`

성공 기준:
- 생성된 기관/관리자 계정이 DB에 존재
- 권한 없는 계정은 위 기능 실행 불가

## 2) 기관 관리자 시나리오

1. 기관 관리자 로그인
2. 부서 엑셀 업로드
3. 직원 엑셀 업로드
4. 템플릿 생성 및 문항 업로드/수정
5. 평가 세션 생성
6. 평가 관계 자동생성 후 수동 보정
7. 세션 시작
8. 진행 중 미제출자 모니터링
9. 세션 종료

성공 기준:
- 단계별 성공/실패 메시지가 UI에 노출
- 실패 이력은 `upload_histories`, 감사로그는 `audit_logs`에 기록

## 3) 일반 사용자 시나리오

1. 사용자 로그인
2. 배정된 평가 목록 진입
3. 임시저장
4. 최종 제출
5. 완료 화면 확인
6. 재제출 불가 시나리오 확인

성공 기준:
- 제출 후 assignment 상태가 `SUBMITTED`
- 재제출 금지 정책 위반 시 명확한 오류 메시지 반환
