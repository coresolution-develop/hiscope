# Stabilization Backlog (First Iteration Draft)

기준일: 2026-03-11
목표: 신규 기능이 아닌 성능/운영 안정성 강화

## P1 (즉시 착수 권장)

1. 실스테이징 자동 리허설 파이프라인 구축
- 내용: Docker+PostgreSQL에서 릴리즈마다 E2E 스모크 자동 실행
- 기대효과: 배포 리스크 사전 차단

2. 운영 대시보드 KPI 표준화
- 내용: 로그인/업로드/제출 성공률, 5xx, 응답시간, 감사로그 적재량 시각화
- 기대효과: 초기 이상 탐지 속도 향상

3. 장애 런북 자동화
- 내용: 헬스체크 실패/오류율 급증 시 알림 + 1차 체크 명령 자동화
- 기대효과: MTTR 단축

## P2 (배포 후 1~2스프린트)

4. 업로드 처리 관측성 강화
- 내용: 업로드 건별 처리시간/실패유형 집계
- 기대효과: 실패 원인 추세 파악

5. DB 성능 점검 주기화
- 내용: 슬로우 쿼리 리포트 정례화 및 인덱스 재검토
- 기대효과: 대량 데이터 성능 저하 예방

6. 감사로그 보존/아카이빙 정책 정교화
- 내용: 보존기간, 압축, 검색 전략 정의
- 기대효과: 감사 추적성과 저장비용 균형

## P3 (중기)

7. 업로드 비동기화/배치 최적화 검토
- 내용: 대량 파일 처리 시 큐 기반 비동기 모델 검토
- 기대효과: 피크 시간대 응답성 개선

8. 권한/격리 보안 회귀 테스트 확장
- 내용: URL 변조/기관 교차 접근 시나리오 확장
- 기대효과: 격리 경계 회귀 위험 감소
