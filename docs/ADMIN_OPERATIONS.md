# 관리자 운영 가이드 (MVP)

## 1. 관리자 유형

- 최고 관리자(`ROLE_SUPER_ADMIN`)
  - 기관 생성/상태 변경
  - 기관 관리자 계정 발급
- 기관 관리자(`ROLE_ORG_ADMIN`)
  - 부서/직원/평가 템플릿/세션/관계/업로드 운영

## 2. 일일 운영 루틴

1. 업로드 이력 확인: `/admin/uploads/history`
2. 평가 세션 상태 확인: `/admin/evaluation/sessions`
3. 진행 중 세션 미제출자 확인: 세션 상세 화면
4. 오류 발생 시 요청 ID 확인 후 로그 조회

## 3. 신규 운영 시작 절차

1. 기관/기관 관리자 계정 생성 (최고 관리자)
2. 기관 관리자가 부서 업로드
3. 기관 관리자가 직원 업로드
4. 평가 템플릿/문항 구성
5. 평가 세션 생성
6. 평가 관계 자동생성/수동보정
7. 세션 시작
8. 진행 중 미제출자 모니터링
9. 세션 종료

## 4. 업로드 운영 기준

- 허용 형식: `.xlsx`
- 크기 제한: 기본 10MB (`APP_UPLOAD_MAX_FILE_SIZE_BYTES`)
- 업로드 실패 시:
  - 화면 메시지 확인
  - 업로드 이력의 실패 상세 확인
  - 동일 파일 재업로드 전 중복 데이터 여부 점검

## 5. 장애 대응 기본

1. 에러 화면의 요청 ID 확인
2. `logs/hiscope-evaluation.log`에서 요청 ID 검색
3. 보안/변경 이벤트는 `logs/hiscope-evaluation-audit.log`와 `audit_logs` 테이블 확인
4. DB migration 실패 시 Flyway 히스토리(`flyway_schema_history`) 점검

## 6. 감사 로그 주요 이벤트

- 로그인 성공/실패
- 기관 생성/상태 변경
- 기관 관리자 계정 생성
- 평가 세션 생성/시작/종료
- 평가 관계 자동생성/수동추가/삭제
- 부서/직원/문항 엑셀 업로드
- 사용자 평가 임시저장/최종제출

## 7. 보안 운영 권고

- 운영에서는 초기 관리자 자동 생성 비활성화 유지
- 관리자 계정은 개인별 발급 및 주기적 비밀번호 변경
- 리버스 프록시/로드밸런서에서 HTTPS 강제
- 외부 공개 시 IP 제한 또는 VPN 적용 권장
