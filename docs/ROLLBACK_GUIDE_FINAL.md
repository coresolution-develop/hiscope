# 롤백 가이드 (최종본)

기준일: 2026-03-11

## 1. 롤백 트리거

아래 중 하나라도 충족하면 롤백 검토를 즉시 시작한다.

- 서비스 기동 실패 또는 헬스체크 연속 실패
- 로그인/권한 경계 붕괴
- 기관 데이터 격리 위반
- 업로드/제출 핵심 플로우 전면 장애

## 2. 초기 대응

1. 신규 배포 트래픽 차단
2. 장애 시점 로그/요청 ID 수집
3. DB 상태 확인 (쓰기 영향 범위 파악)

## 3. 애플리케이션 롤백

1. 직전 안정 버전 이미지/아티팩트 확인
2. 애플리케이션 버전 되돌림
3. 서비스 재기동 후 헬스체크

예시:
```bash
# 직전 버전 태그로 재기동 (환경별 명령에 맞게 적용)
docker compose -f docker-compose.prod.yml up -d
curl -f http://<host>:8099/actuator/health
```

## 4. DB 관련 원칙

- Flyway migration은 일반적으로 자동 다운그레이드를 제공하지 않음
- 스키마 변경 이슈는 백업 복구/포워드 픽스 중 선택
- 데이터 복구는 DBA 승인 하에 수행

## 5. 롤백 후 검증

- 로그인/권한/기관격리 재검증
- 최근 실패 요청 재현 여부 확인
- 감사로그 정상 적재 확인

## 6. 사후 조치

- 장애 원인 분석(RCA)
- 재발 방지 액션 등록
- 다음 배포 전 체크리스트 보강
