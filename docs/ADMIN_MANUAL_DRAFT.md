# 관리자 매뉴얼 초안 (운영 기준)

## 1. 역할과 책임

- 최고 관리자(`ROLE_SUPER_ADMIN`)
  - 기관 생성/상태 관리
  - 기관 관리자 계정 발급
  - 전기관 감사로그 조회
- 기관 관리자(`ROLE_ORG_ADMIN`)
  - 부서/직원/템플릿/세션/평가관계/업로드 운영
  - 기관 단위 감사로그 조회

## 2. 기관 관리자 표준 운영 순서

1. 부서 등록 (수기 또는 엑셀)
2. 직원 등록 (수기 또는 엑셀)
3. 템플릿/문항 구성
4. 평가 세션 생성
5. 평가관계 자동 생성
6. 예외 대상 수동 보정
7. 세션 시작
8. 진행 모니터링 (미제출자 추적)
9. 세션 종료

운영 원칙:
- 세션 시작 전까지만 관계 보정
- 업로드 실패는 이력 확인 후 파일 수정 재업로드
- 오류 발생 시 요청 ID를 기준으로 추적

## 3. 화면별 운영 포인트

- `/admin/uploads/history`
  - 성공/부분성공/실패 상태와 실패 상세 확인
  - 동일 파일 반복 실패 시 데이터 형식/중복키 우선 점검
- `/admin/evaluation/sessions`
  - 세션 상태(PENDING/IN_PROGRESS/CLOSED) 관리
- `/admin/evaluation/sessions/{id}`
  - 진행률/미제출자 모니터링
- `/admin/evaluation/sessions/{id}/relationships`
  - 자동 생성 결과 검증 및 수동 보정
- `/admin/audit`
  - 요청 ID, 액션, 수행자 기준 이슈 추적

## 4. 운영 중 자주 발생하는 이슈 대응

1. 업로드 실패
- 증상: 업로드 후 실패 메시지
- 조치:
  1. 업로드 이력에서 실패 행/필드 확인
  2. 엑셀 데이터 정정
  3. 재업로드

2. 권한 오류(403)
- 증상: 특정 URL 접근 차단
- 조치:
  1. 계정 역할 확인
  2. 기관 소속 확인
  3. 필요 시 최고관리자에게 권한/계정 검토 요청

3. 세션 진행 중 수정 불가
- 증상: 관계/설정 변경 실패
- 조치:
  1. 세션 상태 확인
  2. 정책상 제한 항목인지 확인

4. 사용자 제출 관련 문의
- 증상: 제출/재제출 오류
- 조치:
  1. 세션 상태(진행 중 여부)
  2. 재제출 허용 정책
  3. assignment 상태 확인

## 5. 운영 로그 확인 절차

1. 오류 화면에서 요청 ID 확인
2. 앱 로그에서 요청 ID 검색
3. 감사로그 파일/DB에서 동일 요청 추적
4. 작업자/시각/대상 리소스 확인

## 6. 보안/운영 주의사항

- 운영 기본값으로 슈퍼관리자 자동 생성 비활성화 유지
- 관리자 계정 공유 금지
- 배포 전/후 감사로그 샘플링 점검
- 기관 간 데이터 격리 위반 징후(타 기관 ID 조회 시도) 모니터링
