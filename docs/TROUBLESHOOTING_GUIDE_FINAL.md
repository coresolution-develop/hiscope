# 트러블슈팅 가이드 (최종본)

기준일: 2026-03-11

## 1. 기본 진단 순서

1. 오류 화면/메시지 확인
2. 요청 ID 확보
3. 앱 로그 검색 (`hiscope-evaluation.log`)
4. 감사로그 검색 (`hiscope-evaluation-audit.log`, `audit_logs`)
5. 최근 배포/환경변수 변경 확인

## 2. 자주 발생하는 이슈

## 로그인 실패

점검:
- 계정 상태/비밀번호
- 세션 만료 여부
- 보안 헤더/프록시 설정

## 403 권한 오류

점검:
- 사용자 역할 확인
- 기관 소속 확인
- URL 직접 접근 여부

## 업로드 실패

점검:
- `.xlsx` 형식 여부
- 파일크기/행수 제한 초과 여부
- 필수 컬럼/중복/참조값 오류
- 업로드 이력의 실패 상세

## 제출 불가

점검:
- 세션 상태(IN_PROGRESS 여부)
- 재제출 허용 정책
- assignment 상태

## 3. 감사 추적

필수 확인 필드:
- `request_id`
- `actor_login_id`
- `organization_id`
- `action` / `outcome`

## 4. 긴급 대응

- 다수 사용자 영향 시 즉시 롤백 절차 검토
- 데이터 무결성 의심 시 쓰기 트래픽 차단 후 점검
