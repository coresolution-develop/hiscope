# organization_type 기본 프로파일 검증/보정 (v1.1)

## 1) 병원 기본 Rule Set 검증 결과
- 실제 병원 명부 공통 컬럼 기준 점검 항목:
  - 기관장, 소속장, 부서장, 경혁팀, 경혁팀장, 진료팀장, 1인부서, 평가제외
- 검증 결론:
  - `evaluation_excluded=Y`는 생성 관계에서 제외 처리 필요 (유지)
  - 리더 계층(기관장/소속장/부서장) 규칙을 분리해야 책임 범위가 명확해짐
  - 병원 전용 역할(경혁팀장/진료팀장)에 대한 별도 규칙이 필요
  - `1인부서`는 리더 평가 누락 방지 보완 규칙이 필요

## 2) 계열사 기본 Rule Set 검증 결과
- 실제 계열사 명부 공통 컬럼 기준 점검 항목:
  - 기관장, 소속장, 부서장, 일반직원, 평가제외
- 검증 결론:
  - 병원 전용 속성 없이도 리더 계층 기반 평가는 구성 가능
  - 단일 규칙보다 기관장/소속장/부서장 규칙 분리가 운영 해석에 유리
  - `evaluation_excluded=Y` 제외 규칙은 동일하게 필요

## 3) 보정 포인트
- 기존 v1 대비 보정:
  - 병원: 리더 단일 규칙 → 역할별 다중 규칙 분리
  - 병원: 경혁팀장/진료팀장/1인부서 보완 규칙 추가
  - 계열사: 리더 단일 규칙 → 기관장/소속장/부서장 분리
  - 세트 명을 `v1.1`로 상향

## 4) 병원 기본 Rule Set v1.1 (초안)
1. 기관장 전사 평가 (`DOWNWARD`, priority 10)
2. 소속장 하향 평가 (`DOWNWARD`, priority 20)
3. 부서장 하향 평가 (`DOWNWARD`, priority 30)
4. 경혁팀장-경혁팀 평가 (`CUSTOM`, priority 40)
5. 진료팀장 하향 평가 (`CUSTOM`, priority 50)
6. 1인부서 보완 평가 (`CROSS_DEPT`, priority 60)
7. 의료리더 협업 평가 (`PEER`, priority 70)
- 공통 EXCLUDE: `evaluation_excluded=Y`

## 5) 계열사 기본 Rule Set v1.1 (초안)
1. 기관장 전사 평가 (`DOWNWARD`, priority 10)
2. 소속장 하향 평가 (`DOWNWARD`, priority 20)
3. 부서장 하향 평가 (`DOWNWARD`, priority 30)
- 공통 EXCLUDE: `evaluation_excluded=Y`

## 6) 운영 가이드 분리

### 병원용 업로드/검증/전환
- 업로드 컬럼:
  - 공통: 기관장, 소속장, 부서장, 평가제외
  - 병원 전용: 경혁팀, 경혁팀장, 1인부서, 진료팀장, 의료리더
  - 보조: 평가대상여부, 이전부서명, 입사일자, 퇴사일자
- 검증:
  - 독립 속성 원칙(추론 금지)
  - 표본군 관계 확인: 경혁팀장→경혁팀, 진료팀장→비진료팀장, 1인부서 커버리지
- 전환:
  - LEGACY 비교 → 차이 원인 분류 → override 최소 적용 → RULE_BASED 시작

### 계열사용 업로드/검증/전환
- 업로드 컬럼:
  - 공통: 기관장, 소속장, 부서장, 평가제외
  - 계열사 확장: attr:affiliate_policy_group
  - 보조: 평가대상여부, 입사일자, 퇴사일자, 비고
- 검증:
  - 리더 계층(기관장/소속장/부서장) 관계 적정성 확인
  - 평가제외 누락 여부 확인
- 전환:
  - 리더 규칙 우선 고정 → 비교 검증 → 필요시 세트 복제 후 미세 조정
