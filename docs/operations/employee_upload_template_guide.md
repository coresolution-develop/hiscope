# 직원 업로드 템플릿 가이드

## 기본 컬럼
1. `사원번호(필수)`
2. `이름(필수)`
3. `부서코드(필수)`
4. `직위`
5. `직책`
6. `이메일`
7. `로그인ID(필수)`
8. `상태(ACTIVE/INACTIVE/LEAVE)`

## 병원형 속성 컬럼
- 기본 제공 컬럼
  - `경혁팀(Y/N)`
  - `경혁팀장(Y/N)`
  - `부서장(Y/N)`
  - `1인부서(Y/N)`
  - `진료팀장(Y/N)`
  - `의료리더(Y/N)`
  - `기관장(Y/N)`
  - `소속장(Y/N)`
  - `평가제외(Y/N)`
- 독립 속성 원칙
  - `경혁팀 여부`, `경혁팀장 여부`, `부서장 여부`는 서로 독립 관리
  - `1인부서 여부`, `진료팀장 여부`, `평가제외 여부`도 각각 독립 관리
  - 특정 속성 값으로 다른 속성을 추론하지 않음
  - 예: `경혁팀=Y`, `경혁팀장=Y`, `부서장=N` 유효

## 실제 명부 보조 컬럼(v1)
- `평가대상여부`: v1에서는 업로드 검증용 보조 컬럼으로만 사용 권장 (rule matcher 직접 참조 대상 아님)
- `이전부서명`: v1에서는 이관/검증 참고 정보로만 사용 (관계 생성 규칙 직접 반영 대상 아님)
- `입사일자`, `퇴사일자`: v1에서는 운영 보조 정보로 유지, v2에서 날짜 기반 matcher 확장 포인트로 관리
- 자유 속성 컬럼
  - `attr:clinical_track`
  - `attr:peer_group`
  - 형식: `attr:<attribute_key>` 또는 `속성:<attribute_key>`

## 올바른 입력 예시
- 로그인ID: `nurse001`
- 상태: `ACTIVE`
- boolean 속성: `Y`, `N`, `true`, `false`, `1`, `0`
- 자유 속성 컬럼명: `attr:clinical_track`
- 자유 속성 값: `수술계`, `A군`
- 독립 속성 조합: `경혁팀=Y`, `경혁팀장=Y`, `부서장=N`

## 잘못된 입력 예시
- 로그인ID: `nurse 001` (공백 포함)
- 상태: `WORKING` (허용 상태 아님)
- boolean 속성: `MAYBE`
- 자유 속성 컬럼명: `clinical_track` (`attr:` 접두사 누락)
- 독립 속성 추론: `경혁팀장=Y`이면 `부서장=Y`로 자동 처리된다고 가정

## 운영 팁
- 템플릿의 `가이드` 시트 설명을 먼저 확인
- 업로드 실패 시 오류 메시지의 행/컬럼 기준으로 원본 파일 수정
- 대량 업로드 전 20~50행 리허설 권장
