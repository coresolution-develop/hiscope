# 운영 전 UAT 체크리스트

아래 체크는 기능 확인이 아니라 실제 운영 흐름 검증 기준이다.

## 공통 점검

- [ ] `prod` 프로필에서 앱이 정상 기동된다.
- [ ] `/actuator/health`가 `UP`를 반환한다.
- [ ] 로그인/로그아웃 시 감사로그가 남는다.
- [ ] 에러 화면에 요청 ID가 노출되고 로그에서 추적 가능하다.
- [ ] 권한 없는 페이지 접근 시 403 또는 로그인 리다이렉트가 동작한다.

## 최고 관리자 UAT

- [ ] 기관 생성 후 목록/상세에서 즉시 조회된다.
- [ ] 기관 상태 변경 시 감사로그(`ORG_STATUS_UPDATE`)가 기록된다.
- [ ] 기관 관리자 생성 시 해당 기관으로 정확히 연결된다.
- [ ] 타 권한(기관 관리자/일반 사용자)으로 최고 관리자 기능 접근이 차단된다.

## 기관 관리자 UAT

- [ ] 부서 업로드 성공/부분성공/실패 케이스가 모두 화면과 이력에 반영된다.
- [ ] 직원 업로드 시 중복 로그인 ID/사원번호 오류가 명확히 표시된다.
- [ ] 템플릿/문항 생성-수정-삭제(비활성화)가 정상 동작한다.
- [ ] 세션 생성 후 관계 자동생성/수동추가/삭제가 정상 동작한다.
- [ ] 세션 시작 후 관계 수정이 차단된다.
- [ ] 진행 중 세션 상세에서 미제출 현황이 표시된다.

## 일반 사용자 UAT

- [ ] 본인 배정 목록만 조회된다.
- [ ] 임시저장 후 재접속 시 저장 데이터가 복원된다.
- [ ] 최종 제출 후 완료 화면으로 이동된다.
- [ ] 재제출 불가 세션에서 재수정/중복 제출이 차단된다.
- [ ] 타인의 assignment URL 직접 접근이 차단된다.

## 데이터 격리 UAT

- [ ] 기관 A 관리자는 기관 B 세션/직원/관계 URL 직접 접근 시 차단된다.
- [ ] 업로드/평가/조회 결과에 타 기관 데이터가 혼합되지 않는다.
- [ ] 감사로그의 `organization_id`가 올바르게 기록된다.
