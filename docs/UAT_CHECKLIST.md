# 운영 직전 UAT 체크리스트

기능 존재 여부가 아니라 실제 업무 흐름/운영 증적 기준으로 점검한다.

## 사전 조건

- [ ] `prod` 프로필 또는 prod와 동일한 설정 세트로 UAT 수행
- [ ] 기관 A, 기관 B 테스트 데이터 준비 (데이터 격리 검증용)
- [ ] 최고 관리자/기관 관리자/일반 사용자 계정 준비
- [ ] 감사로그 조회 권한 및 앱 로그 접근 권한 확보

## 공통 운영 품질

- [ ] 로그인/로그아웃/권한거부(403) 각각에서 감사로그가 남는다.
- [ ] 오류 화면에 요청 ID가 표시되고, 앱 로그/감사로그에서 동일 ID로 추적된다.
- [ ] 비정상 요청(잘못된 path variable 타입, 직접 URL 접근)이 500 없이 제어된다.
- [ ] 헬스체크(`/actuator/health`)가 `UP`이며 의존성 오류가 없음을 확인한다.

## 최고 관리자 UAT (기관 운영 흐름)

1. 최고 관리자 로그인
2. 기관 생성
3. 기관 상세 진입
4. 기관 관리자 계정 생성
5. 기관 상태 ACTIVE/INACTIVE 전환
6. 감사로그 조회에서 위 작업 추적

체크 항목:
- [ ] 기관 생성 직후 목록/상세에서 조회된다.
- [ ] 생성된 기관 관리자 계정의 `organization_id`가 해당 기관으로 저장된다.
- [ ] 상태 변경 시 기관 관리자 로그인/업무 접근 정책이 의도대로 동작한다.
- [ ] 감사로그에 `ORG_CREATE`, `ORG_ADMIN_CREATE`, `ORG_STATUS_UPDATE`가 누락 없이 존재한다.

## 기관 관리자 UAT (업무 운영 흐름)

1. 기관 관리자 로그인
2. 부서 업로드(정상 파일)
3. 직원 업로드(정상 파일)
4. 평가 템플릿/문항 구성(수기 + 엑셀)
5. 평가 세션 생성
6. 관계 자동생성 후 수동 보정
7. 세션 시작
8. 진행 현황 모니터링
9. 세션 종료

체크 항목:
- [ ] 업로드 성공/부분성공/실패가 UI 메시지와 업로드 이력에 동일하게 반영된다.
- [ ] 업로드 실패(형식 오류, 행수 초과, 파싱 오류) 시에도 업로드 이력이 남는다.
- [ ] 세션 시작 후 관계 수정 제한이 동작한다.
- [ ] 세션 상세에서 미제출자/진행률이 실제 제출 데이터와 일치한다.
- [ ] 감사로그에서 주요 이벤트가 시계열로 추적 가능하다.

## 일반 사용자 UAT (평가 수행 흐름)

1. 사용자 로그인
2. 본인 배정 목록 진입
3. 평가 작성 후 임시저장
4. 재접속 후 임시저장 복원 확인
5. 최종 제출
6. 완료 화면 확인
7. 재제출 제한/허용 정책 확인

체크 항목:
- [ ] 본인 assignment만 조회된다.
- [ ] 타인 assignment URL 직접 접근 시 차단된다.
- [ ] 제출 후 상태가 `SUBMITTED`로 변경된다.
- [ ] 재제출 불가 세션에서는 수정/제출이 차단된다.

## 기관별 데이터 격리 UAT

- [ ] 기관 A 관리자가 기관 B의 직원/세션/관계/업로드 URL 접근 시 차단된다.
- [ ] 기관 A 업로드 이력에 기관 B 데이터가 섞이지 않는다.
- [ ] 감사로그의 `organization_id`가 실제 수행 주체와 일치한다.
- [ ] 슈퍼관리자 조회 화면에서만 기관 통합 조회가 가능하다.

## 대량 데이터 UAT

- [ ] 직원 목록 페이지네이션(기본 50, 최대 제한) 동작 확인
- [ ] 세션/관계/감사로그 조회 필터 + 페이지네이션 성능 확인
- [ ] 업로드 최대 파일크기/최대 행수 정책 초과 시 즉시 차단 확인
- [ ] 동시 사용자 부하 시 핵심 화면 응답시간/오류율 점검

## 완료 기준 (Go/No-Go)

- [ ] Critical/High 이슈 0건
- [ ] Medium 이슈는 운영 우회/모니터링 방안 포함
- [ ] 체크리스트 증적(스크린샷/로그/DB 쿼리 결과) 확보
